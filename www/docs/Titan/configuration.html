Starting a Titan Node Start a Titan Node by invoking the Java VM with the titan.jar specifying the class to run as sunlabs.titan.node.TitanNodeImpl. /usr/bin/java -cp titan.jar sunlabs.titan.node.TitanNodeImpl file:///var/tmp/titan.cf In production environments, there are other command line options that you might find helpful. For example: /usr/bin/java -Dtitan-node \ -server \ -Xmx12m \ -XX:+UseParallelGC \ -Dcom.sun.management.jmxremote \ -Djava.util.logging.config.file=./etc/logging.properties \ -Dcom.sun.management.jmxremote.password.file=./etc/jmxremote.password.template \ -Dcom.sun.management.jmxremote.port=16001 \ -cp titan.jar sunlabs.titan.node.TitanNodeImpl file:///var/tmp/titan.cf Applications that embed Titan, by creating internal instances of sunlabs.titan.node.TitanNodeImpl or my extending the sunlabs.titan.node.TitanNodeImpl may have other means to start and configure the embedded Titan. Configuring a Titan Node Every instance of Titan (a node) is configured by a set of attribute/value pairs (Configuration Properties). Each Titan node takes a single command-line argument consisting of a URL of a configuration file, the contents of which is a formatted text file containing comments and name/value pairs. For example, invoking Titan in the example above uses the local file /var/tmp/titan.cf as the source of the configuration values. The configuration information could be stored on a web server and the node is capable of using the HTTP protocol to obtain the configuration data. /usr/bin/java -cp titan.jar sunlabs.titan.node.TitanNodeImpl http://webserver.com/titan//node-10.0.1.6-12300-3.cf Configuration Properties If there are no Properties specified in the configuration, each node is configured from a default value. These default values have been chosen arbitrarily chosen (generally).
